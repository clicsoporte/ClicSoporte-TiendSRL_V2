======================================================================
     Guía Técnica: Sistema de Activación de Licencias Híbrido v2.2
======================================================================

Este documento describe el flujo de trabajo y la implementación técnica para activar software cliente (hijos) utilizando Clic-Tools (VPS Vultr) como la Autoridad Central de Licenciamiento.

-----------------------------
1. Resumen del Concepto Híbrido
-----------------------------

El sistema se basa en criptografía asimétrica (RSA) y una arquitectura Cliente-Servidor (API REST) para garantizar que las licencias sean auténticas, revocables y adaptables a diferentes modelos de negocio (por módulos o licencia única).

- **Clic-Tools (El Servidor en Vultr):**
  1. Actúa como la única fuente de verdad (Single Source of Truth).
  2. Gestiona un catálogo de diferentes programas hijos (ID_Software).
  3. Emite "Tokens de Activación" temporales que vinculan una compra con un hardware específico.
  4. Provee un Endpoint seguro para "Heartbeats" (latidos) que validan la vigencia en tiempo real.

- **Software Hijos (Los Clientes):**
  1. Poseen la Clave Pública del servidor (incrustada u ofuscada).
  2. Modo Online: Se activan vía API y reciben un JSON firmado y un JWT de sesión.
  3. Modo Offline: Si falla la red, validan la última firma guardada localmente durante un periodo de gracia.

-----------------------------
2. Implementación en Clic-Tools (Servidor)
-----------------------------

La lógica principal reside en el módulo de licencias, ahora extendido para soporte API.

-- Archivo: `src/modules/licenses/lib/crypto.ts` --
(Maneja RSA-SHA256 para firmar los payloads que se envían a los hijos)

-- Archivo: `src/modules/licenses/lib/db.ts` --
(Maneja el catálogo de software y las activaciones por HWID)

-----------------------------
4. Plan de Acción REFINADO v2.2: Autoridad Central y Modularidad
-----------------------------

### Fase 1: Catálogo de Software y API en Vultr
- **Registro de Programas:** Interfaz en Clic-Tools para dar de alta cada software hijo (ej: Clic-Cola, Clic-POS, Clic-Cloud).
- **Endpoint de Activación (`/api/v1/activate`):**
  - Entrada: `softwareId` + `activationToken` + `hardwareFingerprint`.
  - Salida: JSON de licencia firmado + JWT de sesión de 24h.

### Fase 2: Granularidad por Módulos (Flags)
- **Feature Flags:** En el servidor, el administrador marca qué módulos están activos para esa licencia específica.
- **Validación en el Hijo:** El software hijo lee el campo `"features"` del JSON firmado para habilitar o bloquear menús y funciones internamente sin necesidad de versiones de código distintas.

### Fase 3: JWT y Heartbeats (Control Online)
- **Token de Sesión:** Tras la activación, el servidor entrega un JWT.
- **Heartbeat (Latido):** El hijo envía este JWT cada 4-12 horas.
- **Kill-Switch:** Si el administrador revoca la licencia en el VPS, el siguiente Heartbeat devolverá un estado de "Revocado", borrando la licencia local y bloqueando el programa.

### Fase 4: Resiliencia Offline (Periodos de Gracia)
- **Cache Local:** El JSON firmado se guarda en el disco del cliente.
- **Contador de Desconexión:** Si el hijo no puede alcanzar el VPS de Vultr (sin internet), permite el uso basado en la cache local por un máximo de 7 días. Al 8vo día, exige conexión para continuar.

### Fase 5: Protección Avanzada (Anti-Tampering)
- **Fingerprinting Profundo:** El `hardwareId` se genera combinando: Serial de Placa Base + UUID de Partición de Sistema + Nombre de PC.
- **Time-Drift Protection:** El servidor envía su hora actual en cada Heartbeat. Si la diferencia con el reloj local es > 1 hora, el hijo sospecha de fraude de tiempo y pide sincronización.

-----------------------------
5. Flujo de Venta para el Usuario Final
-----------------------------
1. El cliente compra el software.
2. El administrador genera un **Token de Activación** en Clic-Tools (VPS).
3. El cliente instala el software, digita el token.
4. El software se conecta al VPS, registra su hardware y queda activado.
5. Si el cliente no tiene internet, el administrador le envía el archivo `license.json` generado manualmente desde el portal de Clic-Tools.
