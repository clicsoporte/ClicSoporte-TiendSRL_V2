======================================================================
     Guía Técnica: Sistema de Activación de Licencias Offline
======================================================================
v2.0 - Implementación Segura

Este documento describe el flujo de trabajo y la implementación técnica para activar software cliente (como Clic-Cola) de forma offline, utilizando Clic-Tools como la autoridad de licenciamiento. Esta versión incluye un método más seguro que dificulta la manipulación por parte del usuario final.

-----------------------------
1. Resumen del Concepto (Seguro)
-----------------------------

El sistema se basa en criptografía asimétrica (un par de claves pública/privada) y en ocultar los artefactos de licenciamiento para prevenir la manipulación.

- **Clic-Tools (El Servidor):**
  1. Posee una **Clave Privada** secreta y única en el servidor.
  2. Al generar una licencia, empaqueta la información (ID de hardware, fechas, etc.) en un objeto JSON.
  3. Utiliza su Clave Privada para "firmar" digitalmente esta información.
  4. Genera un archivo `license.json` que contiene los datos y la firma.
  5. El administrador descarga este archivo para entregárselo al cliente.

- **Clic-Cola (El Software Cliente - Implementación Segura):**
  1. La **Clave Pública** que corresponde a la clave privada del servidor está **incrustada directamente en el código fuente** como una constante de texto. No existe un archivo `public_key.pem` visible.
  2. En la pantalla de activación, el cliente tiene un botón **"Importar Licencia"**. Al usarlo, selecciona el archivo `license.json` que le proporcionó el administrador.
  3. La aplicación lee el contenido del `license.json` y lo **guarda en su propia base de datos interna SQLite**. El archivo `.json` original ya no es necesario.
  4. Al arrancar, Clic-Cola lee los datos de la licencia desde su base de datos interna (no desde un archivo).
  5. Usa la Clave Pública (incrustada en su código) para verificar la firma. Si la firma es válida, significa que el contenido es auténtico.
  6. Procede a verificar que el `hardwareId` coincida y que la licencia no haya expirado. Si todo es correcto, el software se activa.

Este método es mucho más seguro porque el usuario no puede simplemente cambiar la clave pública (está compilada en la aplicación) ni el archivo de licencia (está guardado en una base de datos).

-----------------------------
2. Implementación en Clic-Tools (El Servidor)
-----------------------------

La lógica principal para firmar la licencia reside en `src/modules/licenses/lib/crypto.ts` y `src/modules/licenses/lib/db.ts`.

-- Archivo: `src/modules/licenses/lib/db.ts` --
```typescript
// Fragmento de la función que crea una licencia
export async function addLicense(licenseData: Omit<License, 'id' | 'createdAt' | 'licenseKey'>): Promise<License> {
    const db = await connectDb(LICENSES_DB_FILE);
    
    // 1. Recopilar la información que irá dentro de la licencia.
    const licenseInfo = {
        softwareId: licenseData.softwareId,
        clientCompanyId: licenseData.clientCompanyId,
        hardwareId: licenseData.hardwareId,
        isPerpetual: licenseData.isPerpetual,
        expirationDate: licenseData.expirationDate,
        createdAt: new Date().toISOString(),
    };

    // 2. Llamar a la función crypto para firmar estos datos y generar el JSON.
    const signedLicenseJson = await signLicenseData(licenseInfo);

    // 3. Guardar el JSON firmado completo en la base de datos de Clic-Tools.
    const info = db.prepare(`
        INSERT INTO licenses (licenseKey, /*... otros campos ...*/)
        VALUES (@licenseKey, /*... otros valores ...*/)
    `).run({
        licenseKey: signedLicenseJson,
        // ... resto de los datos ...
    });

    const result = db.prepare('SELECT * FROM licenses WHERE id = ?').get(info.lastInsertRowid) as License;
    return result;
}
```

-----------------------------
4. Implementación en Clic-Cola (El Cliente) - GUÍA PARA EL OTRO GEMINI
-----------------------------

El software cliente debe ser capaz de importar el `license.json` y validar su autenticidad de forma segura.

**Paso 1: Incrustar la Clave Pública**
- Crea un archivo `src/lib/keys.ts`.
- Dentro, exporta la clave pública (descargada desde Clic-Tools) como una constante de texto.
-- Archivo: `src/lib/keys.ts` --
```typescript
// La clave pública se incrusta directamente para dificultar su manipulación.
export const publicKey = `-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...
... (pegar aquí el contenido completo de public_key.pem) ...
-----END PUBLIC KEY-----`;
```

**Paso 2: Almacenar la Licencia en la Base de Datos Local**
- Modifica la base de datos de Clic-Cola (`database/db.ts`) para incluir una tabla de configuración (`settings`) con una columna `licenseData TEXT`.
- Crea acciones en `src/actions/licenseActions.ts` para guardar y leer esta información.
```typescript
// En src/actions/licenseActions.ts
export async function saveLicenseData(data: string): Promise<void> {
    const db = await connectDb();
    db.prepare(`UPDATE settings SET licenseData = ? WHERE id = 1`).run(data);
}

export async function getLicenseData(): Promise<string | null> {
    const db = await connectDb();
    const result = db.prepare(`SELECT licenseData FROM settings WHERE id = 1`).get();
    return result?.licenseData || null;
}
```

**Paso 3: Implementar el Validador de Licencia**
- El siguiente es un ejemplo de cómo se podría implementar el validador en `src/lib/license-validator.ts`.

```typescript
import crypto from 'crypto';
import os from 'os';
import { publicKey } from './keys'; // Importa la clave incrustada
import { getLicenseData } from '../actions/licenseActions'; // Importa la función de la BD

// Genera un ID de hardware único para la máquina actual.
const generateHardwareId = async (): Promise<string> => {
    const hostname = os.hostname();
    const platform = os.platform();
    const arch = os.arch();
    return `${hostname}-${platform}-${arch}`;
};

interface ValidationResult {
    isValid: boolean;
    message?: string;
}

export async function validateLicense(): Promise<ValidationResult> {
    try {
        // 1. Leer los datos de la licencia desde la base de datos interna.
        const licenseFileContent = await getLicenseData();
        if (!licenseFileContent) {
            return { isValid: false, message: 'No se encontró una licencia importada.' };
        }
        
        const licenseData = JSON.parse(licenseFileContent);
        const { license_info, signature } = licenseData;

        // 2. VERIFICAR LA FIRMA DIGITAL con la clave pública incrustada.
        const messageToVerify = JSON.stringify(license_info, Object.keys(license_info).sort());
        const verifier = crypto.createVerify('RSA-SHA256');
        verifier.update(messageToVerify);
        const isSignatureValid = verifier.verify(publicKey, signature, 'hex');

        if (!isSignatureValid) {
            return { isValid: false, message: 'Firma de licencia inválida. El archivo puede estar corrupto o alterado.' };
        }

        // 3. VERIFICAR EL HARDWARE ID
        const currentHardwareId = await generateHardwareId();
        if (license_info.hardwareId !== currentHardwareId) {
            return { isValid: false, message: `La licencia no es válida para esta máquina. ID de máquina: ${currentHardwareId}` };
        }

        // 4. VERIFICAR LA FECHA DE VENCIMIENTO
        if (!license_info.isPerpetual && new Date(license_info.expirationDate) < new Date()) {
            return { isValid: false, message: 'La licencia ha expirado.' };
        }

        return { isValid: true };

    } catch (error) {
        console.error("Error validando la licencia:", error);
        return { isValid: false, message: 'Error inesperado al validar la licencia.' };
    }
}
```

**Paso 4: Integrar en la Interfaz de Activación**
- En la pantalla de activación de Clic-Cola, añade un botón **"Importar Archivo de Licencia"**.
- Al hacer clic, abre un diálogo para seleccionar el archivo `license.json`.
- Lee el contenido del archivo y guárdalo en la base de datos usando `saveLicenseData()`.
- Llama a `validateLicense()` y activa la aplicación si la validación es exitosa.
- Si no, muestra el mensaje de error junto con el ID de hardware actual para que el usuario pueda solicitar una nueva licencia.
