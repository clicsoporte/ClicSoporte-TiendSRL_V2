
# GUÍA DE DESPLIEGUE EN VPS (UBUNTU 24.04 LTS) - CLIC-TOOLS
# Arquitectura: Node.js + Next.js + SQLite + PM2 + Nginx + SSL

Esta guía detalla los pasos para configurar un VPS de 1 vCPU / 1GB RAM para hospedar el sistema de forma segura y eficiente.

---

## 1. ESTIMACIÓN DE CONSUMO DE RECURSOS
### Escenario: 2 Técnicos Activos / 10 Clientes (Uso Bajo)

Con la arquitectura actual (Server Actions + SQLite), el consumo estimado es:

*   **MEMORIA RAM (Runtime):** ~450MB - 550MB totales.
    - OS (Ubuntu): ~200MB.
    - App (Node.js/Next.js): ~250MB.
    - Nginx + PM2: ~50MB.
    - *Nota:* El 1GB de RAM física es suficiente para la ejecución, pero el SWAP de 2GB es VITAL para el proceso de `npm run build` (compilación).
*   **CPU (vCPU):** < 2% en reposo. Spikes de 10-15% durante la generación de PDFs o sincronización masiva con el ERP.
*   **ALMACENAMIENTO (SSD):** ~6GB ocupados.
    - OS + Dependencias: ~5GB.
    - Base de Datos (10 clientes): < 50MB (crece muy lento).
    - Quedan ~19GB libres para backups y archivos adjuntos.
*   **ANCHO DE BANDA:** Consumo extremadamente bajo por ser mayoritariamente texto y JSON.
    - *Advertencia:* El límite de 0.16GB mencionado es inusualmente bajo para un VPS (típicamente son 1TB+). Si realmente es ese el límite, evita descargar backups pesados o subir muchas fotos de alta resolución en los proyectos.

---

## 2. PREPARACIÓN INICIAL DEL SISTEMA

### 2.1. Actualizar repositorios
sudo apt update && sudo apt upgrade -y

### 2.2. Configuración de Memoria Virtual (SWAP) - OBLIGATORIO
Dado que el servidor tiene 1GB de RAM, Next.js fallará al compilar sin SWAP. Crearemos 2GB de memoria virtual.

sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Hacerlo permanente tras reinicios
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

---

## 3. INSTALACIÓN DEL ENTORNO DE EJECUCIÓN

### 3.1. Instalar Node.js (Versión 20 LTS o superior)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

### 3.2. Instalar Gestores de Procesos y Herramientas
sudo npm install -g pm2
sudo apt install -y git build-essential

---

## 4. DESPLIEGUE DE LA APLICACIÓN

### 4.1. Clonar y Preparar (Ejemplo usando Git)
cd /var/www
# git clone <tu_repositorio> clic-tools
# cd clic-tools

### 4.2. Configurar Variables de Env
# Crear el archivo .env.local con tus credenciales de SQL Server (ERP)
nano .env.local

### 4.3. Instalar Dependencias y Compilar
npm install
npm run build

### 4.4. Iniciar con PM2 (Gestión de Procesos)
pm2 start npm --name "clic-tools" -- start

# Configurar para que inicie automáticamente al bootear el VPS
pm2 startup
# (Ejecutar el comando que imprima la pantalla)
pm2 save

---

## 5. SERVIDOR WEB Y SEGURIDAD (REVERSE PROXY)

### 5.1. Instalar Nginx
sudo apt install -y nginx

### 5.2. Configurar el Sitio en Nginx
sudo nano /etc/nginx/sites-available/clic-tools

# Pegar la siguiente configuración:
server {
    listen 80;
    server_name tu-dominio.com; # Cambiar por tu IP o dominio

    location / {
        proxy_pass http://localhost:9003; # Puerto configurado en package.json
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

### 5.3. Activar Configuración y Reiniciar Nginx
sudo ln -s /etc/nginx/sites-available/clic-tools /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

---

## 6. CERTIFICADO SSL (HTTPS) - GRATUITO Y AUTOMÁTICO

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d tu-dominio.com

---

## 7. FIREWALL (UFW) - ENDURECIMIENTO DE SEGURIDAD

# Permitir SSH (Asegúrate de no quedarte fuera)
sudo ufw allow ssh

# Permitir Tráfico Web
sudo ufw allow 'Nginx Full'

# Activar Firewall
sudo ufw enable

---

## 8. PERSISTENCIA DE DATOS (NOTAS IMPORTANTES)

1. La base de datos SQLite reside en la carpeta `dbs/intratool.db`. 
2. Es altamente recomendable hacer backups periódicos de esta carpeta hacia un almacenamiento externo.
3. Si el ERP (SQL Server) está en tu oficina, debes añadir la IP de este VPS al "Whitelisting" del firewall de tu oficina para el puerto 1433.
